# 房屋中介管理系统

一个基于 C 语言实现的课程设计项目，面向“房屋中介业务管理”场景，覆盖房源管理、租客注册登录、看房预约、租约签订、审核统计、数据持久化等完整流程。项目采用命令行交互，支持 Windows 与 Linux/macOS 终端运行，适合课程答辩、功能演示和基础软件工程实践。

## 项目简介

系统围绕三类角色展开：

- 管理员：负责维护中介、租客、房源、分类信息、统计报表与系统数据
- 中介：负责提交房源、处理预约、发起租约、维护业务记录
- 租客：负责注册登录、筛选房源、预约看房、查看合同、处理待签合同

系统默认使用本地二进制文件 `rental_data.dat` 持久化数据，并在大多数写操作后自动保存，适合单机课程项目场景。

## 主要功能

### 1. 管理员端

- 中介信息管理：新增、修改、删除、查询中介
- 租客信息管理：查询、修改、删除租客
- 房源信息管理：新增房源、修改房源、删除房源、组合查询、排序
- 房源审核：审核中介提交的待审核房源，支持通过或驳回
- 看房与租约管理：查看看房记录、查看租约记录、管理租约履约状态
- 分类管理：维护区域、楼层说明、朝向、房型、装修等分类项
- 数据维护：手动保存、从默认文件恢复、备份到指定文件、从指定文件恢复
- 系统维护：重置中介/租客密码、修改管理员密码、生成演示数据
- 信息统计：房源统计、看房统计、租约统计、租客租房统计、按月签约统计

### 2. 中介端

- 新增租房合同，发起待签约流程
- 支持从“已完成看房”关联发起合同，自动填充房源/租客信息
- 新增“合同工作台”：支持将已确认看房标记完成，并从已完成看房一键发起合同
- 修改合同履约状态、删除合同、查看本人经手租约
- 查询与排序本人相关的预约、租约、房源
- 查看本人负责看房记录，修改或删除看房记录
- 查询房源时段可预约性
- 提交新房源进入待审核流程
- 处理待处理预约，支持同意/拒绝
- 查看本人提交房源的审核状态，并支持重新提交
- 修改个人密码
- 查看个人业务统计

### 3. 租客端

- 注册账号，系统自动分配租客 ID
- 通过租客 ID 或手机号登录
- 查询房源、筛选房源并发起看房预约
- 修改或删除未完成的看房预约
- 查看我的看房预约、我的租房合同
- 填写看房反馈
- 修改个人资料
- 处理待签合同，支持确认签订或拒绝
- 待签合同可填写拒绝理由；确认后自动联动房源状态与看房合同状态

合同推荐操作路径：

1. 租客预约看房
2. 中介在“处理待处理预约”中同意
3. 中介在“合同工作台”将该看房标记为已完成
4. 中介在“合同工作台”从已完成看房发起合同
5. 租客在“处理待签合同”中确认/拒绝

说明：合同ID（租约ID）由系统自动分配，无需手工输入。
说明：合同工作台中的“看房预约ID”指预约记录ID（如 3001/3002），不是房源ID（如 2001/2002）。

- 查看个人信息统计

## 核心业务流程

### 房源流程

1. 管理员可直接录入并上架房源
2. 中介新增房源时先进入“待审核”状态
3. 管理员审核通过后房源进入可预约状态；审核不通过则记录驳回原因

### 看房预约流程

1. 租客查询房源并选择时间发起预约
2. 系统校验预约时间必须晚于当前时间
3. 系统校验房源时段和中介时段冲突，避免重复预约
4. 中介处理待处理预约，形成正式看房安排
5. 看房完成后租客和中介可分别补充反馈

### 租约流程

1. 中介基于房源与租客信息发起租房合同
2. 合同先进入“待签订”状态
3. 租客在个人菜单中确认签订或拒绝
4. 已签订合同可继续维护履约状态，包括有效、到期、提前退租
5. 系统会根据合同状态刷新房源可租状态

## 业务规则与约束

- 房源存在待签合同或生效中的有效合同时，不允许再次预约或重复发起合同
- 看房预约时间必须是未来时间
- 已完成看房记录不允许随意删除
- 未签订合同不能直接修改为履约状态
- 多次登录失败会触发账号锁定，降低暴力尝试风险
- 多个程序实例并行使用时，关键菜单和关键写入流程会先同步默认数据文件，尽量降低数据覆盖问题

## 数据存储与安全设计

### 数据持久化

- 默认数据文件为 `rental_data.dat`
- 当前持久化格式为稳定的 `v6` 二进制结构
- `v6` 使用逐字段、小端序写入，避免结构体填充差异导致的数据不兼容
- 保留了对旧版 `v2/v3/v4/v5` 数据文件的兼容读取能力
- 大多数写操作后会自动保存，也支持管理员手动保存和备份恢复

合同增强说明：

- 租约记录已扩展合同字段（关联看房ID、租期、押金、其他条款、拒签原因）
- 看房记录新增合同关联状态（未发起/待确认/已完成）
- 管理员强制终止合同会写入 `contract_ops.log` 操作日志

### 认证与密码

- 密码不再以明文形式持久化，保存时使用带盐哈希
- 旧数据中的明文密码在加载后会自动迁移为哈希格式
- 登录连续失败 5 次后，账号会锁定 15 分钟
- 管理员未登录状态下不支持密码找回
- 中介和租客找回密码、管理员重置密码时，系统会生成临时密码
- 使用临时密码登录后，建议立即在个人菜单中修改为自己的新密码

说明：本项目的认证与存储机制适合课程设计演示，不应直接视为生产级安全方案。

## 构建与运行

### 编译

在项目目录执行：

```bash
gcc -Wall -Wextra -Wpedantic -std=c11 -Iinclude \
  src/app/main.c \
  src/core/rental_system.c src/core/storage.c \
  src/modules/demo_data.c src/modules/bootstrap_data.c \
  src/modules/password_utils.c src/modules/domain_utils.c \
  src/modules/login_guard.c src/modules/data_path_utils.c \
  -o rental_system
```

Linux/macOS 可直接运行：

```bash
./rental_system
```

Windows (MinGW) 可执行：

```bash
gcc -Wall -Wextra -Wpedantic -std=c11 -Iinclude ^
  src/app/main.c ^
  src/core/rental_system.c src/core/storage.c ^
  src/modules/demo_data.c src/modules/bootstrap_data.c ^
  src/modules/password_utils.c src/modules/domain_utils.c ^
  src/modules/login_guard.c src/modules/data_path_utils.c ^
  -o rental_system.exe
```

Windows (MSVC Developer Command Prompt)：

```bat
cl /W4 /std:c11 /I include ^
  src\app\main.c ^
  src\core\rental_system.c src\core\storage.c ^
  src\modules\demo_data.c src\modules\bootstrap_data.c ^
  src\modules\password_utils.c src\modules\domain_utils.c ^
  src\modules\login_guard.c src\modules\data_path_utils.c ^
  /Fe:rental_system.exe
```

### CMake 构建

适用于安装了 CMake 的环境：

```bash
cmake -S . -B build
cmake --build build
```

运行可执行文件：

```bash
./build/rental_system
```

Windows 下运行：

```bat
build\rental_system.exe
```

说明：

- 若环境没有 CMake，请直接使用上面的 `gcc` 全编译命令。
- 两种方式产物等价，均可用于课程演示。

### 关于 .vscode

- `.vscode` 目录仅用于本地 VS Code 便捷调试与补全，不是项目构建所必需。
- 团队成员或评审同学可不使用 VS Code，直接通过 `cmake` 或 `gcc` 命令完成编译与运行。
- 本仓库已在 `.gitignore` 中忽略 `.vscode/`，避免将本地编辑器配置提交到版本库。

### 运行

```bash
./rental_system
```

首次启动时，若默认数据文件不存在，系统会自动初始化基础数据并生成演示数据。

## 交互说明

- 主菜单支持管理员登录、中介登录、租客登录、租客注册、保存/恢复数据、密码找回、生成演示数据
- 多数输入位置支持 `#` 回退上一级
- 某些数值输入场景中，`-1` 在不属于当前合法范围时也可用于回退
- 租客注册成功后，系统会显示新分配的租客 ID，需要自行记录以便后续登录

## 项目结构

- `include/`：公共头文件目录
- `src/app/main.c`：程序入口
- `src/core/`：核心业务与存储实现
- `src/modules/`：按功能拆分的辅助模块（演示数据、密码、校验、登录保护、路径处理）
- `CMakeLists.txt`：CMake 构建脚本
- `rental_data.dat`：默认业务数据文件
